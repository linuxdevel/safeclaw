# Better CLI/TUI (Ink-based)

> SafeClaw v2 — Feature 5 (depends on Feature 4: Streaming UX)

## Problem

The current readline CLI is a single scrolling terminal with no visual structure. We want OpenCode-style split panes with agent output, chat input, and a status sidebar.

## Design Decisions

- **Ink (React for terminals)**: mature, composable, great for complex layouts. Adds react + ink as dependencies.
- **New package**: `packages/tui/` keeps TUI separate from existing CLI. CLI becomes a thin wrapper.
- **Fallback mode**: `--no-tui` flag uses the existing readline adapter.
- **UsageTracker**: new core component to track token usage and cost across providers.

## Keyboard shortcuts

| Shortcut | Action |
|----------|--------|
| `Ctrl+C` | Exit |
| `Ctrl+X` then `↓` | Enter sub-agent view mode |
| `←` / `→` in sub-agent view | Switch between sub-agents |
| `↑` in sub-agent view | Back to main agent |
| `Ctrl+L` | Clear screen |
| `Ctrl+P` | Command palette |

### Task 2: UsageTracker (core)

**File**: `packages/core/src/agent/usage-tracker.ts`

```typescript
interface UsageStats {
  promptTokens: number;
  completionTokens: number;
  totalTokens: number;
  estimatedCostUsd: number;
  model: string;
  maxContextTokens: number;
}

class UsageTracker {
  record(model: string, promptTokens: number, completionTokens: number): void;
  getStats(): UsageStats;
  reset(): void;
}
```

Cost calculation uses a rate table:
```typescript
const COST_PER_1K: Record<string, { prompt: number; completion: number }> = {
  "claude-sonnet-4": { prompt: 0.003, completion: 0.015 },
  "claude-opus-4": { prompt: 0.015, completion: 0.075 },
  "gpt-4o": { prompt: 0.0025, completion: 0.01 },
  // etc.
};
```

**Test**: `packages/core/src/agent/usage-tracker.test.ts`

### Task 3: Agent emits usage events

**File**: `packages/core/src/agent/agent.ts`

After each model provider call, record usage in the UsageTracker. Add UsageTracker to AgentConfig (optional, created by default).

The streaming events (`AgentStreamEvent`) gain a new event type:
```typescript
{ type: "usage", promptTokens: number, completionTokens: number }
```

### Task 4: App component

**File**: `packages/tui/src/components/App.tsx`

Root component that renders the three-region layout:
- Uses Ink's `<Box>` with `flexDirection="row"` for main + sidebar
- Main area uses `flexDirection="column"` for output + input
- Manages keyboard shortcuts via `useInput` hook
- Holds state: current view mode (main/sub-agent), active sub-agent index

### Task 5: AgentOutput component

**File**: `packages/tui/src/components/AgentOutput.tsx`

Scrollable region showing agent responses:
- Renders markdown to Ink components (bold, code blocks, headers)
- Tool execution shown inline: `→ Running: read(...)` / `✓ completed`
- Auto-scrolls to bottom on new content
- Manual scroll with mouse wheel or keyboard

### Task 6: StatusPanel component

**File**: `packages/tui/src/components/StatusPanel.tsx`

Right sidebar showing:
- Token usage: `45K / 128K (35%)`
- Cost: `$0.12`
- Model name
- Modified files list (tracked via tool results from write/edit)
- Active tool indicators
- Sub-agent list with status (when Feature 6 is implemented)

### Task 7: ChatInput component

**File**: `packages/tui/src/components/ChatInput.tsx`

Bottom region:
- Multi-line text input (Enter to send, Shift+Enter for newline)
- Command history (up/down arrows)

### Task 8: TUI adapter

**File**: `packages/tui/src/adapter.ts`

`TuiAdapter` implements `ChannelAdapter` interface, wired to the Ink components. Bridges between the agent's message/streaming interface and React component state.

### Task 9: CLI integration

**File**: `packages/cli/src/cli.ts`

Add `--no-tui` flag. Default behavior launches TUI. With flag, falls back to existing readline adapter.
